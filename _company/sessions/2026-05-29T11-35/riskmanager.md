### 🛡️ 최종 목표: Quantitative Safety Protocol V3.0 확정
**목표:** 모든 실패 시나리오 발생 시, 시스템이 자동으로 진입하는 가장 보수적이고 방어적인 포지션 사이징 및 거래 정지 조건을 수치적으로 정의합니다.

#### 1. 리스크 원칙 재확인 (계좌잔고 $C$ 기준)
*   **단일 포지션 최대 허용 손실:** $0.5\% C$ (기존 2% $\to$ 스트레스 테스트 기반 보수적 축소)
*   **동시 오픈 포지션 합산 최대 허용 손실:** $1.5\% C$ (기존 5% $\to$ 극도의 방어 모드 적용)
*   **자동 중단 트리거 기준:** 단일 시나리오 발생 시, 예상 최악의 경우 $2.0\% C$ 초과 손실이 예측되면 즉시 모든 거래를 정지합니다.

#### 2. 실패 시나리오별 리스크 임계값 정의 (Critical Thresholds)
각 실패 시나리오는 단순히 코드 오류가 아니라, **'최대 예상 손실(Worst-Case Loss)'**을 의미합니다. 이에 대응하는 자동 방어 로직을 설계해야 합니다.

| 시나리오 | 발생 원인/유형 | 리스크 매니저 판단 (최악의 경우) | 강제 안전 조치 (Safety Action) |
| :---: | :--- | :--- | :--- |
| **S001** | 데이터 누락(Null/NaN) | 데이터 기반 트레이딩 자체가 불가능하므로, 포지션 크기 $\mathbf{0}$으로 설정. 최대 손실 예측액 $= \mathbf{0}$. | 🚫 신호 생성 즉시 차단 (Status Code 400). 진입 전 $1$회 이상 데이터 유효성 체크 필수. |
| **S002** | 속도 제한 초과/데이터 순서 오류 | 시장 상황 판단 자체를 할 수 없게 되므로, 모든 포지션 리스크 제로화 필요. 최대 손실 예측액 $= \mathbf{0}$. | 🚫 거래 전송 즉시 차단 (Status Code 503). $1$분 간격으로 데이터 스트림 체크 및 대기 모드 진입. |
| **S003** | 과도한 변동성/이상치 발생 | ATR 기반 리스크 관리 자체가 무의미해지므로, 시장에 대한 베팅을 포기해야 함. 최대 손실 예측액 $= \mathbf{Max Drawdown\ Breached}$. | 🚫 진입 허용 규모를 $\text{Position Size} = 0$으로 강제 설정 (Status Code 429). $N$-시그마 임계값 초과 시 무조건 정지. |

#### 3. 산출물 생성
이 기준들을 모아 최종 시스템에 적용할 프로토콜 문서를 작성합니다. 이 파일은 모든 에이전트의 테스트 결과가 통합되어야 하는 최상위 문서입니다.

## 1. 목적
시스템 스트레스 테스트 환경에서 발생하는 모든 실패 시나리오에 대응하여, 포지션 사이징 및 거래 실행을 강제적으로 중단시키기 위한 정량적 안전 기준(Threshold)을 확정한다. 본 프로토콜은 시스템 Go-Live 직전 최종 방어선 역할을 수행하며, 어떠한 경우에도 계좌의 안정성을 최우선으로 한다.

## 2. 핵심 리스크 원칙 (기준점: 계좌잔고 $C$)
*   **단일 포지션 최대 허용 손실:** $\mathbf{0.5\% \cdot C}$
    *   (이전 기준 대비 보수적 축소율 적용)
*   **동시 오픈 포지션 합산 최대 허용 손실:** $\mathbf{1.5\% \cdot C}$

## 3. 실패 시나리오별 자동 방어 로직 (Failure Mode Detection & Response)

| 시나리오 코드 | 발생 감지 조건 (Trigger) | 리스크 매니저 판단 | 시스템 강제 동작 (Action/Status Code) |
| :---: | :--- | :--- | :--- |
| **S001** | 데이터 유효성 검증 실패 (Null, NaN 등) | 트레이딩 근거 부재. 포지션 크기 $\mathbf{0}$ 강제 적용. | Status Code 400 (Bad Request). 신호 생성 전 $N$개 이상 연속 Null 값 감지 시 무조건 차단. |
| **S002** | 스트리밍 속도/순서 오류 (Rate Limit, Out-of-Sequence) | 시장 정보의 시간적 완전성 훼손. 포지션 리스크 $\mathbf{0}$ 강제 적용. | Status Code 503 (Service Unavailable). 데이터 수신 간격이 $T_{max} = [1 \text{분}]$을 초과하거나, 순서 오류가 발생하면 즉시 거래 중단 및 대기 모드 진입. |
| **S003** | 과도한 변동성/이상치 (Extreme Volatility) | ATR(Average True Range) 기반 리스크 산출 자체가 무효화됨. 포지션 크기 $\mathbf{0}$ 강제 적용. | Status Code 429 (Too Many Requests). 현재 $ATR$ 값이 사전 정의된 임계값 $(ATR_{threshold} = [1.5] \cdot \text{최근 } ATR)$을 초과할 경우, 무조건 거래 정지 및 시장 관망 모드 진입. |

## 4. 최종 승인 조건
본 프로토콜은 모든 에이전트가 개발한 시스템의 **'안정성 테스트(Stress Test)'**를 성공적으로 통과해야만 효력을 발휘한다. 각 시나리오별로 위에 명시된 상태 코드가 정확하게 발생하고, 이에 따라 포지션 크기가 $\mathbf{0}$으로 설정되는 것을 모든 로그에서 확인해야 한다.
