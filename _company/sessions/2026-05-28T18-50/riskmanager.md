### 🚨 FINAL\_SAFETY\_PROTOCOL V2.0 최종 교차 검증 체크리스트

**검토 목표:** `trading_engine.py`에 통합된 모든 안전 로직이 정의된 리스크 기준과 일치함을 입증하고, 시스템 활성화의 최종 승인 자료를 제공한다.

#### 1. 게이트키퍼 기능 (Gatekeeper Logic Verification)
| 항목 | 요구사항 (기준) | 현재 구현 상태 (코드 확인 필요) | 검토 결과 | 조치/지적 사항 |
| :--- | :--- | :--- | :--- | :--- |
| **A. 진입 전 리스크 체크** | 단일 포지션 최대 리스크(1~2%) 및 계좌 대비 초과 여부를 *주문 실행 직전* 반드시 검증해야 함. | `SafeModeManager`의 승인 로직이 `execute_trade()` 시작 시 호출됨. (✅) | **PASS** | 📐 계산식 명시: $P_{size} \le (\text{Account Balance} \times \text{Risk\%})$ |
| **B. 리스크 합산 체크** | 동시 오픈 포지션의 누적 위험도가 계좌 대비 5%를 초과하는 경우, 모든 진입 시도를 차단해야 함. | 이 로직이 `SafeModeManager` 내에서 구현되었는지 확인 필요. | **PENDING** | 🚫 만약 합산 리스크 검증이 코딩되지 않았다면, 이는 치명적 결함입니다. 즉시 추가 요청합니다. |
| **C. 손절 임계점 강제 적용** | 모든 포지션은 진입과 동시에 유효한 손절가(Stop Loss)를 필수적으로 정의해야 함. (손절선 없는 진입 원천 차단) | `signal` 입력값에 SL 값이 포함되어야만 로직이 진행되도록 수정 필요. | **CONFIRM** | 코드 레벨에서 'SL 값 부재 시'는 트레이드 승인 불가로 강제합니다. |

#### 2. 비상 상황 대처 절차 (Emergency Protocol Verification)
| 항목 | 요구사항 (기준) | 현재 구현 상태 (코드 확인 필요) | 검토 결과 | 조치/지적 사항 |
| :--- | :--- | :--- | :--- | :--- |
| **A. Level 3 발생 시** | 시스템은 즉시 모든 트레이딩을 중단하고, 자동 관제(Manual Control) 모드로 전환하며, 사용자에게 알림을 전송해야 함. (Hard Shutdown) | `SafeModeManager`가 Level 3 트리거를 감지하면 강제 종료시키는 코드가 작동하는지 테스트 필요. | **VERIFY** | 이탈 방지를 위해 하드웨어 레벨의 안전 스위치(Kill Switch) 구현 여부를 확인합니다. |
| **B. 연속 손실 임계값 (5회)** | 5연속 손실 감지 시, 시스템은 자동적으로 트레이딩 권한을 상실하고 경보를 발령해야 함. | `SafeModeManager`의 상태 변수(`ConsecutiveLossCount`)가 정확히 관리되고 있는지 코드로 확인 필요. | **VERIFY** | 이 카운터는 오작동 시 전체 봇 정지를 유발하므로, 테스트 케이스가 가장 중요합니다. |
