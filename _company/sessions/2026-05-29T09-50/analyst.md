## 1. 목표 및 목적
본 문서는 트레이딩 봇 시스템의 포지션 사이징 및 자산 배분을 결정하는 핵심 모듈(Dynamic Portfolio Module)에 필요한 실시간 시장 데이터를 표준화하고, 이를 코다리 개발팀에 전달하기 위함이다. 모든 데이터 입력은 리스크 매니저가 정의한 '실시간 위험 경보 레벨'과 상호 연동되어야 한다.

## 2. 필수 계산 지표 및 구조
| 지표 | 설명 | 산출 주기 (Frequency) | 필요한 원본 데이터 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **Covariance Matrix ($\Sigma$)** | 포트폴리오 내 자산들 간의 공분산 행렬. 분산 및 상관관계를 계산하는 기반. | 5분봉 단위 (최소) / 실시간 스트리밍 | BTC, ETH, 주요 Altcoin N개 종목의 5분봉 OHLCV 데이터 | 최소 20개 거래 쌍에 대한 $\Sigma$ 필요. |
| **Correlation Matrix ($R$)** | 자산 간의 선형 상관관계 ($\rho$). $\Sigma$로부터 계산됨. | 1시간 단위 (재계산) | 동일 | $R_{ij} = \frac{\text{Cov}(i, j)}{\sigma_i \sigma_j}$ 공식 적용. |
| **Volatility Vector ($\sigma$)** | 각 자산의 개별 변동성. 주로 표준편차 기반. | 1시간 단위 (재계산) | 종가(Close Price) 시퀀스 | 이동표준편차 (Rolling Std Dev) 사용 권장. |
| **Market Beta ($\beta_M$)** | 포트폴리오의 시장 민감도 측정 지표. 전체 시장 대비 노출도를 판단. | 1시간 단위 (재계산) | BTC/SPY(혹은 전체 코인 인덱스) 가격 데이터 | $\text{Cov}(Asset, Market) / \sigma_{Market}^2$ 공식 적용. |

## 3. 데이터 전송 및 포맷 규격
*   **데이터 타입:** JSON (JavaScript Object Notation) 형식 권장. 빠르고 파싱하기 용이함.
*   **전송 빈도:**
    *   `Covariance Matrix`: 최소 5분마다 업데이트하여 실시간 변동성을 반영한다.
    *   `Correlation/Volatility`: 시장 구조 변화가 감지되는 시점(예: 1시간 단위 마감 또는 VIX 급등)에 맞춰 재계산 및 전송하는 것이 효율적이다.
*   **구조 예시 (JSON Payload):**

```json
{
  "timestamp": "YYYY-MM-DDTHH:MM:SSZ",
  "status_code": 200, // 200 OK / 400 Data Error 등
  "cov_matrix": {
    "BTC": {"BTC": 1.0, "ETH": 0.85, ...},
    "ETH": {"BTC": 0.85, "ETH": 1.0, ...},
    // ... (N개 자산)
  },
  "correlation_matrix": {
    "BTC": {"BTC": 1.0, "ETH": 0.85, ...},
    // ...
  },
  "volatility_vector": {
    "BTC": 0.025, // 표준편차 (예: 2.5%)
    "ETH": 0.031  // 표준편차
  }
}
```

## 4. 통합 연동 및 예외 처리 (Safety Gate)
*   **데이터 누락 시나리오:** 특정 자산의 데이터가 일정 시간 이상(예: 20분) 수신되지 않을 경우, 시스템은 즉시 포지션 사이징을 중단하고 '경고 레벨 🟠'으로 격상한다.
*   **시스템 재부팅/API 장애:** 데이터 전송 실패 시, 이 모듈의 계산 결과는 무효 처리되어야 하며, 트레이딩 엔진은 이전 안정화된 값 또는 안전 모드(Safe Mode)로 전환된다. 이는 리스크 매니저가 정의한 `Shutdown Protocol`에 따라 동작한다.
