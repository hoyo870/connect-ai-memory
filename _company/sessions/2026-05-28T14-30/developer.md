### II. 시스템 운영 프로세스 상세 체크리스트 (Technical Checklist)

**A. Safe Mode Manager 프로토콜 통합 시나리오:**
1. **API Rate Limit 초과 감지:** `async_api_manager` 내에서 연속적인 API 호출 실패(429 Too Many Requests)가 N회 이상 발생할 경우, 시스템은 즉시 '경고 레벨'로 전환되어 모니터링만 수행한다. (Action: 1차 방어선)
2. **시스템 과부하 감지:** 트레이딩 엔진의 CPU 사용률이 임계치 T%를 초과하거나 메모리 누수(Memory Leak)가 의심되는 경우, 시스템은 '제한 레벨'로 전환된다. (Action: 2차 방어선)
3. **시장 이상 상태 감지:** 시장 변동성 지표(Volatility Index)가 평소 대비 M 표준편차를 벗어나거나, 전일 종가 대비 급격한 괴리가 발생할 경우, 모든 트레이더 에이전트의 진입 신호 생성 기능은 자동으로 정지된다. (Action: 최종 방어선 - Safe Mode 활성화)

**B. 필수 기술 검증 목록 (Technical Verification Checklist):**
*   [ ] API Rate Limit 초과 시 자동 감지 및 15분간 주문 실행 중단 로직 구현 완료.
*   [ ] 비동기 데이터 파이프라인의 트랜잭션 무결성(ACID) 검증 필수.
*   [ ] Safe Mode 진입/탈출 과정에서 모든 상태 변경 이력(State Change Log)을 기록하는 Audit Trail 시스템 확보.
