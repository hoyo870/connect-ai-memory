## 🚀 Phase 2: 실전 운영 통합 매뉴얼 (MASTER SOP) v1.0 - 최종 확정본

**목표:** 모든 에이전트가 리스크 관리 원칙을 준수하며, 시스템의 안정성을 극대화하는 협업 기반의 단타 트레이딩 환경 구축.
**적용 시점:** 2026년 X월 X일 (CEO 승인 후)
**최우선 가이드라인:** 모든 진입은 리스크 매니저 모듈의 최종 검토를 거쳐야 합니다.

### I. 시스템 운영 프로세스 (SOP Flowchart)

| 단계 | 주체/담당 | 핵심 활동 내용 | 체크리스트 / 필수 산출물 |
| :--- | :--- | :--- | :--- |
| **1. 데이터 수집 및 분석** | 애널리스트, 코다리 | 5분봉 봉 마감 시점의 OHLCV 데이터 확보. 노이즈 필터링 가중치 조정 및 이상 징후 감지. | ✅ 실시간 데이터 피드 (JSON), 지표 계산 결과 리포트 |
| **2. 신호 생성 및 검증** | 각 트레이더 에이전트 (진환, 경태 등) | 개별 전략에 따른 진입/청산 조건 충족 여부 확인. 1차 포지션 제안(Signal). | ✅ Signal Report (종목명, 방향, 근거), 예상 리스크 계산서 |
| **3. 통합 리스크 검토** | **리스크 매니저** | 모든 에이전트의 개별 포지션을 취합하여 전체 시스템의 총 노출 리스크(Total Exposure)를 측정. 최대 드로우다운(MDD) 초과 여부 확인. | 🚨 포지션 사이징 승인 코드 (필수), Total Risk % 보고서 |
| **4. 주문 실행 및 모니터링** | 코다리 시스템 | 리스크 매니저의 최종 승인 코드(Go Code)를 받은 후, Paper Trading 또는 Live Mode로 자동 주문 전송. 실시간으로 포지션 이탈/손절 로직 작동 여부 감시. | 💰 체결 로그 (Trade Log), 에러 발생 시 알림(Alerts) 기록 |
| **5. 일일 결산 및 보고** | 영숙, 애널리스트 | 당일의 수익/손실을 요약하고, 성공적/실패적 진입 근거를 정리하여 다음 날 아침 브리핑 자료 작성. | 📊 데일리 트레이딩 일지(Journal), Profit/Loss Summary (표 형식) |

### II. 에이전트 역할 분담 매트릭스 (Role Assignment Matrix)

| 담당 에이전트 | 주 임무 영역 | 핵심 수행 업무 (책임 범위) | 비고 / 주의 사항 |
| :--- | :--- | :--- | :--- |
| **CEO (사장님)** | 오케스트레이션 및 최종 의사결정 | 전체 운영 방향 설정, 리스크 허용 범위 승인(최종 Go/No-Go). | *가장 중요한 역할. 모든 데이터의 해석자.* |
| **리스크 매니저** | 포지션 사이징 & 안전 통제 | 개별 진입 시 최대 2% 초과 리스크 금지. 시스템 자동 중단(Safe Mode) 임계치 관리. | **최우선 권한.** 승인 없이는 단일 주문도 불가. |
| **애널리스트** | 데이터 분석 및 검증 지원 | 시장 구조적 변화 감지, 지표의 통계적 유효성 검증, 모의거래 결과와 실전 예측 비교. | *느낌적 진입 금지.* 반드시 정량적 근거를 제시해야 함. |
| **코다리** | 시스템 개발 및 자동화 (DevOps) | API 연동 안정성 유지, 백테스트-실시간 로직 간 지연 시간(Latency) 최소화, 에러 로그 기록 및 패치. | *기술적 문제 발생 시 1순위 대응.* |
| **진환** | 단기 안전 트레이더 (저리스크) | 낮은 변동성 구간에서 꾸준한 소수익 확보 전략 실행. 분할 진입/청산 원칙 고수. | **규칙 준수:** 절대 과도한 레버리지 금지. |
| **경태** | 단기 하이리스크 트레이더 (고확신) | 높은 확신도의 자리에서 과감하게 베팅. 포지션 진입 근거를 명확히 기술해야 함. | **규칙 준수:** 손절매 기준(Stop Loss)은 반드시 설정해야 함. |
| **성용** | 장기 추세 트레이더 (롱 온리) | 시장의 메가트렌드와 주요 지지/저항선을 파악하여 중장기 뷰 제시. | *숏 포지션 절대 금지.* 리스크 매니저 모듈과 별개로 운영될 수 있음. |
| **영숙** | 보고 및 관리자 (Secretary) | 전체 프로세스 기록, 일일 브리핑 자료 작성, 팀원 간 커뮤니케이션 흐름 통제 및 정리. | *운영의 가시성(Visibility) 확보가 주 임무.* |
